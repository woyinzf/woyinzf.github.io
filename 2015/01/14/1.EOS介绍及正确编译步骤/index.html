EOS简介：

EOS.IO 软件引入一种新的区块链架构设计，它使得去中心化的应用可以横向和纵向的扩展。这通过构建一个仿操作系统的方式来实现，在它之上可以构建应用程序。该软件提供帐户、身份验证、数据库、异步通信和跨越数百个CPU内核或集群的应用程序调度。由此产生的技术是一种区块链架构，它可以扩展至每秒处理百万级交易，消除用户的手续费，并且允许快速和轻松的部署去中心化的应用。

摘自：EOS.IO 技术白皮书

引入一个小插曲，本人前期参与研究go-ethereum的的联盟链搭建，pow，pbft，istanbul，clique共识的研究及solidtiy的编写。因为go-ethereum性能问题，TPS亲测大概30~40左右，后期拜读各显神通的白皮书，靠谱的太少，因为EOS的作者曾参与了由BTS,Steem的研发，三者都采用了著名的graphene技术(实现高TPS的关键技术)，目前BTS的TPS在100k左右，EOS号称百万TPS, 因此带着合法的兴奋剂来转战研究EOS.进入正题。

编译环境：

Ubuntu 17.10

编译步骤：

1.下载代码，编译代码（可以建立自己的目录）

git clone https://github.com/eosio/eos –recursive

cd eos
git checkout dawn-2.x
./build.sh ubuntu

2.启动eos，连接到eos测试节点（手动对应好目录）

cd ~/eos/build/scripts
./start_npnode.sh

执行后，一般提示如下
Synching requires at least 8 minutes, depending on network conditions.

3.查看同步情况（我的大概同步一小时，具体因网速决定，中间报错可以忽略，直到同步完毕）

tail -F testnet_np/stderr.txt

成功提示：

sudo netstat -ntlp
如有如下提示：（已经跑起）
tcp 0 0 127.0.0.1:8888 0.0.0.0: LISTEN 3988/../programs/eo
tcp 0 0 0.0.0.0:5355 0.0.0.0: LISTEN 637/systemd-resolve
tcp 0 0 0.0.0.0:9877 0.0.0.0:* LISTEN 3988/../programs/eo

端口8888：是eos的rest监听端口。

端口9877：是eos的p2p节点互联端口。
