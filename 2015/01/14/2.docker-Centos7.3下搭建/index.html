2.docker 环境搭建及测试

By Zhaofa Yin

2015-01-14 更新日期:2018-01-22

文章目录

++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++
system: Centos7.3
版本信息：
Docker version 1.12.6

client：105
server：103

++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++

1.安装docker：
yum install docker

2.服务端：下载registry 镜像，及运行镜像：
docker pull registry

docker run -d -v /opt/docker/registry:/var/lib/registry -p 5000:5000 –restart=always –name registry registry:latest

3.客户端下载镜像及上传镜像：
（通过将镜像push到registry来验证，首先将主机的registry镜像命名为符合仓库要求registry_url:port/dir/name:version的格式，通过docker tag命令来实现命名）

docker pull mysql

docker tag docker.io/mysql:latest 172.16.192.103:5000/mysql:1.1

docker push 172.16.192.103:5000/mysql:1.1

4.上传不成功处理：

修改客户端：（重启客户端，重新推送）
在”/etc/docker/“目录下，创建”daemon.json“文件。在文件中写入：

{ “insecure-registries”:[“172.16.192.103:5000”] }

服务端：
以上不可以再执行此步骤：
查看SELinux状态： (修改为关闭)
a、/usr/sbin/sestatus -v ##如果SELinux status参数为enabled即为开启状态
SELinux status: enabled
关闭SELinux：
b、临时关闭（不用重启机器）：
set enforce 0

测试实例：==================================redis测试：

1.容器的操作：

docker pull docker.io/redis

docker run -d –name service-redis -p 6379:6379 redis

docker run -it –link service-redis:redis –rm redis redis-cli -h redis -p 6379

redis 127.0.0.1:6379> set name zhangsan

　　设置一个key-value对。
　　查询数据
　　redis 127.0.0.1:6379> get name
　　“zhangsan” 　　

2.镜像的操作：

创建镜像：
docker tag docker.io/redis:latest 172.16.192.103/my_redis/docker.io/redis:2.6

推送镜像：
docker push 172.16.192.103/my_redis/docker.io/redis:2.6

获得镜像：
docker pull 172.16.192.103/my_redis/docker.io/redis:2.6
